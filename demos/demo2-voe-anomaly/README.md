# Demo 2: VoE 物理異常偵測

**最快出成果的 PoC** — 利用 LeWM 的 Violation-of-Expectation 機制，偵測場景中的物理異常事件。

## 展示效果

```
正常狀態                          異常瞬間
┌──────────────────┐            ┌──────────────────┐
│                  │            │   📦 掉落！       │
│  📦 靜止在桌上    │            │     ↓             │
│  ──────          │            │  ──────          │
│  機台正常運轉     │            │  機台正常運轉     │
│                  │            │                  │
│ Surprise: 0.02   │            │ Surprise: 0.87   │
│ ████░░░░░░░░░░░  │            │ █████████████░░  │
│ [正常]           │            │ [⚠️ 物理異常!]   │
└──────────────────┘            └──────────────────┘
```

## 核心原理

```
時間 t:   Encoder(frame_t) → z_t
          Predictor(z_t, action_t) → ẑ_{t+1}   (LeWM 預測的下一步)

時間 t+1: Encoder(frame_{t+1}) → z_{t+1}       (實際觀測)

Surprise = MSE(ẑ_{t+1}, z_{t+1})

if Surprise > θ:
    觸發「物理異常」警報
```

這就是 LeWM 論文中 VoE 的核心——世界模型預期「物體應該繼續靜止」，但實際觀測到「物體在移動」，產生高 surprise。

## 為什麼這比傳統異常偵測更強

| 方法 | 能偵測什麼 | 不能偵測什麼 |
|------|-----------|-------------|
| YOLO | 特定類別（安全帽、人、車） | 未訓練過的物件行為 |
| 背景差分 | 任何移動的東西 | 靜態異常、緩慢變化 |
| LeWM VoE | **任何違反物理預期的事件** | 視覺正常但語義異常的情況 |

## 展示場景（建議順序）

### 場景 A: 物體掉落（最容易）
- 桌上放一個物件，訓練時它一直靜止
- 展示時把它推落 → surprise 飆升
- 展示：LeWM 知道「物體不該突然移動」

### 場景 B: 人員跌倒
- 訓練資料中人都正常行走
- 展示時模擬跌倒 → surprise 飆升
- 展示：不需要專門的「跌倒偵測」模型

### 場景 C: 設備異常
- 訓練資料中機台正常運轉（固定節奏）
- 展示時模擬異常振動或停機 → surprise 飆升
- 展示：不需要針對每種設備訓練專用模型

### 場景 D: 顏色變化 vs 物理變化（論文經典對比）
- 改變場景光線/顏色 → surprise 低（外觀改變但物理正常）
- 物體瞬移/掉落 → surprise 高（物理異常）
- 展示：LeWM 理解「物理」而非只看「像素」

## 實作步驟

### Phase 1: 環境準備 (Week 1)
- [ ] 設定固定攝影機拍攝場景
- [ ] 拍攝 2-4 小時「正常場景」影片
- [ ] 轉換為 LeWM 訓練格式（HDF5）
- [ ] action 設為零向量或光流 pseudo-action

### Phase 2: 訓練 LeWM (Week 1-2)
- [ ] 在正常場景資料上訓練 LeWM
- [ ] 驗證 prediction loss 收斂
- [ ] 在 held-out 正常資料上確認 surprise 基線值

### Phase 3: 異常偵測管線 (Week 2-3)
- [ ] 實作即時 surprise 計算管線
- [ ] 設定 surprise 閾值 θ（取正常資料的 mean + 3σ）
- [ ] 實作 surprise 時間序列即時繪圖 UI
- [ ] 測試各種異常場景的 surprise 反應

### Phase 4: UI 與展示 (Week 3-4)
- [ ] 即時畫面 + surprise 條 + 警報
- [ ] 錄製 A/B 對比影片
- [ ] 調校閾值（避免誤報）

## 關鍵實驗

### 實驗 1: Surprise 分佈
- 正常場景的 surprise 分佈（應為低值、窄分佈）
- 各種異常場景的 surprise（應明顯高於正常）
- 畫 histogram 確認可分離性

### 實驗 2: 不同異常類型的敏感度
- 快速事件（物體掉落）vs 緩慢事件（設備漸漸偏移）
- 大幅異常 vs 微小異常
- 視覺改變 vs 物理改變

### 實驗 3: 閾值選擇
- ROC 曲線：不同 θ 下的 TPR vs FPR
- 找到最佳操作點

## 成功指標

- [ ] 正常 vs 異常的 surprise 分佈可明確分離（p < 0.01）
- [ ] 物體掉落偵測率 >= 95%
- [ ] 誤報率 < 5%（正常場景不觸發警報）

- [ ] 端到端延遲 < 30ms
